## 2. 背景与目标

**业务背景**：
用户是二次元周边（谷子）购买者，经常参与多个拼团购买。随着参团数量增加，难以追踪每个团的到货状态、已交金额、囤货截止时间等信息。现有工具（备忘录、Excel）不够直观，缺乏图片关联和状态管理。

**产品目标**：
- 让用户能直观地记录和管理所有谷子购买记录
- 按团维度聚合，清晰展示每个团的财务和到货状态
- 通过数据可视化，让用户了解自己的消费趋势

**用户画像**：
- 二次元文化爱好者，经常购买动漫/游戏周边
- 参与多个拼团，需要管理复杂的购买记录
- 偏好可爱、二次元风格的界面
- 移动端使用为主

## 3. 范围定义

### 3.1 包含范围 (In Scope)
- 卡片创建、编辑、删除
- 图片上传与展示
- 按团名分类浏览
- 团看板（累计金额、状态统计）
- 个人统计面板与图表
- 数据导出/导入（JSON）

### 3.2 不包含范围 (Out of Scope)
- 后端账号系统（V1阶段，预留接口）
- 多用户协作
- 支付功能
- 消息通知

### 3.3 MVP边界
- **Must**：卡片CRUD、按团分类、基础统计、localStorage持久化
- **Should**：图表可视化、数据导入导出
- **Could**：主题切换、动画效果

## 4. 数据模型

### 4.1 卡片 (Card)

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | string | 唯一标识 | UUID |
| image | string | 图片 | base64 或 URL |
| name | string | 谷名 | 1-20字 |
| price | number | 价格 | ≥0 |
| group | string | 团名 | 非空 |
| status | enum | 谷子状态 | pending/disbanded/arrived/shipped |
| deadline | string | 囤货DDL | 日期格式 |
| note | string | 备注 | 可选 |
| createdAt | string | 记录时间 | ISO日期，默认可修改 |
| updatedAt | string | 更新时间 | ISO日期 |

### 4.2 状态映射

| 显示名称 | 存储值 | 颜色标识 |
|----------|--------|----------|
| 未到货 | pending | 橙色 |
| 已散 | disbanded | 灰色 |
| 已到货 | arrived | 绿色 |
| 已排发 | shipped | 蓝色 |

## 5. 页面结构

### 5.1 底部导航
- 首页（图标：首页/列表）
- 我的（图标：用户/统计）

### 5.2 首页

**全部Tab**：
- 按添加时间倒序展示所有卡片
- 卡片展示：图片、谷名、价格、状态标签

**团名Tab**（动态生成，每个已录入的团一个Tab）：
- 团看板（Tab下方固定区域）：
  - 累计金额：该团「已散」状态以外的所有卡片金额总和
  - 状态分布：未到货X / 已到货X / 已排发X / 已散X
- 卡片列表：该团下的所有卡片

**添加按钮**：右下角悬浮按钮，点击进入新建卡片页

### 5.3 新建/编辑卡片页

**表单字段**：
1. 图片上传：点击区域上传，支持预览
2. 谷名：文本输入，1-20字限制
3. 价格：数字输入，展示为金额
4. 团名：文本输入 + 下拉选择（从历史团名中选择）
5. 谷子状态：单选/分段选择器（未到货、已散、已到货、已排发）
6. 囤货DDL：日期选择器
7. 备注：多行文本输入
8. 记录时间：日期选择器，默认当前时间，可修改

### 5.4 我的页面

**顶部统计面板**（4个指标卡片）：
1. X月吃谷：当前月份添加的卡片数量 → 点击展示当月所有卡片
2. X月交肾：当前月份添加的卡片金额总和 → 点击展示当月每日金额折线图
3. 食量：累计卡片数量（全部历史） → 点击展示每月卡片数量折线图
4. 肾款：累计金额总和（全部历史） → 点击展示每月金额折线图

**团列表**：
- 按团名分组展示
- 每行：团名 | 已到货X | 未到货X | 已排发X
- 点击团名 → 跳转到首页对应Tab

## 6. 交互细节

### 6.1 首页交互
- Tab左右滑动切换
- 卡片长按/点击展开更多操作（编辑、删除）
- 下拉刷新（手势或按钮）

### 6.2 卡片操作
- 点击卡片 → 进入编辑页
- 左滑/长按 → 显示删除按钮
- 删除需二次确认

### 6.3 统计面板交互
- 点击指标卡片 → 底部弹出详情面板
- 详情面板内展示对应列表或图表
- 点击遮罩或关闭按钮收起

## 7. 数据持久化策略

### V1（Demo阶段）
- 使用 localStorage 存储所有卡片数据
- 提供「导出数据」功能：生成JSON文件下载
- 提供「导入数据」功能：上传JSON文件恢复
- 数据格式版本号管理，便于后续迁移

### V2（后续）
- 接入后端账号系统
- 用户通过唯一标识登录
- 数据云端同步
- 支持跨设备、跨浏览器

## 8. 设计方向

- **风格**：二次元可爱风、亮色系
- **主色调**：彩虹渐变、粉色系点缀
- **字体**：圆润可爱
- **图标**：手绘风格或圆润icon
- **动效**：柔和过渡、微弹性

## 9. 技术方案

- 纯前端实现：HTML + CSS + JavaScript
- 单页应用（SPA），Hash路由
- 图表库：Chart.js（CDN引入）
- 图片处理：FileReader转base64
- 存储：localStorage

## 10. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| localStorage容量限制 | 中 | 高 | 监控容量，提供导出清理 |
| 图片base64过大 | 高 | 中 | 限制图片大小，提示压缩 |
| 数据误删 | 中 | 高 | 删除确认，导出备份 |
